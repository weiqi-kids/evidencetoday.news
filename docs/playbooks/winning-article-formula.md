# Playbook：能贏的文章模子（Winning Article Formula）

> **這是本站選題與寫文章的第一順位方法論。** 撰寫或改寫任何一篇 `articles`（也適用 `ingredients`／`myths` 的選題思路）之前，**先讀這份**，再進 `content-guide.md` 的區塊結構規範。
>
> 一句話：**在低權威網域上，你贏不了「權威決定的頭部詞」，但你能贏「具體、急迫、乏人問津、且限定台灣情境的健康問題」。** 我們能贏的問題，恰好就是這個站該存在來回答的問題——策略與存在意義在此收斂。

---

## 為什麼要有這份 playbook（勝績解剖）

實測數據（2026-07，GSC 近 28 天）給了一個乾淨的對照。兩篇**同主題、同作者、同網域權威**的褪黑激素文章：

| | 🏆 `import-melatonin-taiwan-customs` | 💤 `melatonin-prescription-taiwan-gray-market` |
|---|---|---|
| 表現 | 排名 **p9.5**、吃掉全站 **66%** 點擊 | 174 曝光、只 **2** 點擊、排名很深 |
| 回答幾個問題 | **一個**：帶回台灣合法嗎？ | 一大堆：是不是處方藥＋劑量＋時差vs失眠＋六國比較＋灰色市場 |
| 讀者狀態 | 「我**現在**行李箱有幾瓶／正要下單」＝當下要做的決定 | 「我想了解褪黑激素這件事」＝泛泛好奇 |
| 誰在跟你搶 | 沒人——權威站不寫「海關規定」 | 所有人——「褪黑激素是什麼」人人在寫 |

**結論**：差別不在權威（兩篇一樣低），在**題目的性質**。Google 長尾排名獎勵「對某一個具體問題意圖，答得最完整、最貼身」的頁。贏家把相關性密度堆到最高（每個 H2 都在答同一問題的一個面向、來源全是官方），輸家把相關性稀釋在五個主題上，對任何單一查詢都不是最佳解。

**這不只是 SEO 技巧，是本站存在意義的操作化。** 讀者真正的痛不是資訊稀缺，而是健康搜尋結果幾乎每一條都在賣東西、嚇人、或套用不適用台灣的美國語境。本站的價值，是在「切身、當下、做錯有後果」的決定時刻，給一個「不賣你東西、講台灣、對你說實話（包括坦白說不知道）」的可信判斷。贏家文之所以贏，跟這個站之所以該存在，是**同一個原因**。

## 鎖定參數：一篇能贏的文章必備「六基因」

一個題目要在**我們這個網域現在**就有機會贏，必須**同時**具備六個基因。缺任何一個，就會退化成上表那種「輸家」泛論文。寫之前先逐條自問，六個都是「是」才動筆：

1. **單一具體決定**——讀者此刻必須做的**一個**選擇（能不能帶／要不要停／會不會衝突／該選哪個），**不是**主題導覽或百科式介紹。
2. **「現在」觸發點**——有明確的搜尋時刻：正要出國／正要下單／正要開刀／正要抽血健檢／剛拿到一種藥。文章對應一個真實情境，不是「總有一天可能想知道」。
3. **台灣在地限定**——法規、在地買得到什麼、台灣人的真實情境。**這是護城河**：美國語境或通用大站結構上答不了「帶回**台灣**」「**台灣**買得到嗎」。缺這條，就是在跟全世界高權威站正面打。
4. **權威站沒在寫的角度**——他們寫頭部詞（成分是什麼、有什麼功效、劑量怎麼算），不寫這個具體縫隙。**主動避開權威決勝的戰場。**
5. **切身後果**——法律／安全／金錢，重到讓人願意主動搜、願意讀完（代購最重 7 年、術前不停可能術中出血、檢驗數值被驗錯導致誤判）。
6. **答案先行＋範圍狠收**——每個 H2 一句粗體結論再給證據；FAQ 對應真人子問題；官方/一級來源；**把相鄰主題明確踢給別篇**（贏家文開頭就講「該不該吃是另一篇的事」）。

> 記憶口訣：**「一個台灣人此刻正要做、做錯有後果、而網路上只查得到美國語境或業配答案的健康決定。」**

## 選題法：去哪裡找這種題目

不要從「這個成分我還沒寫」出發（那會寫成百科）。從「**台灣人此刻正要做的健康決定**」出發。幾個高產的題型家族（都是 melatonin-customs 的同構複製）：

- **進口／合法性／在地買得到嗎**：某成分在美國是保健品、在台灣是藥品或列管，代購普遍、乏人正解。例：NMN 在台灣合法嗎、CBD 帶回台灣、紅麴保健品在台灣。
- **保健品×藥物的交互作用（有「現在」觸發）**：讀者剛拿到一種藥、或家人在吃某種藥。例：魚油和抗凝血藥、紅麴和 statin、Q10 和降血脂藥。
- **某個關卡前要不要停／要準備什麼**：手術、健檢抽血、懷孕、出國。例：開刀前要停哪些保健品、抽血前 biotin 要停嗎。
- **兩個看似一樣、選錯有後果的東西怎麼選**：讀者站在貨架前要做決定。例：某兩種規格／劑型差在哪、哪種適合我這種情況。

**與「擴寫牽引力叢集」的關係**：這份 playbook 和「優先擴寫已有曝光的主題叢集」（如 melatonin）不衝突，而是它的原理。叢集之所以能被擴大，正因為每一篇都是一個 winning-formula 形狀的具體問題。擴叢集時，**每個子題都要獨立通過上面六基因**，不要為了補齊叢集而寫一篇泛論。

**數據輔助**：`node scripts/audience-insights.mjs` 的三桶、以及排名 5–15 有曝光的 query，可用來確認需求真實存在；但需求訊號只是輔助，題目形狀（六基因）才是決定能不能贏的主因。

## 撰寫結構（接 content-guide 的 AEO 區塊，不重複）

版型與 frontmatter schema 一律照 [`../content-guide.md`](../content-guide.md)（30 秒重點 / BLUF / FAQ / references 規範、禁 AI 模板開頭）。這裡只補「贏家形狀」特有的三點：

- **開頭第一句＝把讀者拉進那個具體情境**（一個真實畫面或一個直接主張），並在前兩段就講清楚「本篇只回答哪一個問題、哪些相鄰問題請看別篇」。範圍收窄是贏家的核心動作。
- **每個 H2 是同一問題的一個面向**，不是換一個主題。自問：把這個 H2 拿掉，讀者對「那一個決定」的判斷會不會缺一塊？不會就刪。
- **來源優先序**：涉及法規→政府機關（食藥署／海關／全國法規資料庫）；涉及臨床→一級文獻或國際指引＋台灣在地醫療媒體/醫院衛教各至少一。每筆寫進 frontmatter 結構化 `references`。

## 常見陷阱

- **寫成百科／主題導覽**（最常見的退化）：一篇想把某成分「講完整」。症狀＝H2 之間主題跳躍、readingTime 偏長、標題是名詞短語而非問句。解法＝拆成數篇各自具六基因的具體問題文，彼此內鏈。
- **缺「台灣在地」基因**：寫成放諸四海皆準的通用衛教，等於回到跟高權威站正面打。每篇都要有「在台灣，這件事是……」的在地錨點。
- **為擴叢集而稀釋**：補子題時寫了一篇六基因不全的湊數文，反而稀釋叢集權威。寧缺勿濫（YMYL）。
- **後果講得太輕**：沒有把「做錯會怎樣」講到讓人有動機讀完，就抓不住那個搜尋時刻。

## 驗證清單（動筆前 + 發布前）

**動筆前（選題 gate）**
- [ ] 六基因逐條為「是」？（單一決定／現在觸發／台灣在地／權威沒寫／切身後果／可窄可答）
- [ ] 一句話說得出「哪一個台灣人、在什麼時刻、正要做什麼決定」？
- [ ] 標題是**讀者會打的問句**，不是名詞短語？
- [ ] 相鄰主題已規劃好踢給哪些現有/未來篇（設定內鏈）？

**發布前（沿用既有 gate）**
- [ ] `pnpm content:audit` 全綠（無 banned-opening / ai-phrase / vague-reference）
- [ ] `pnpm build` 零錯誤
- [ ] 每個 H2 都在回答「那一個問題」的一個面向，沒有主題漂移
- [ ] `references` 每筆含可點 `url`，法規/臨床來源優先序正確
- [ ] `relatedArticles`/`relatedIngredients` 雙向內鏈，把讀者導到叢集其他篇
