# 本日有據 Corporate Identity 使用規範

## 1. 品牌定位

本日有據是健康知識與健康議題編輯平台，核心定位如下：

- 以研究證據、公共資訊與主編判讀整理健康議題。
- 用一般人能理解的語言說明。
- 幫助讀者建立健康觀念與判斷力。
- 整體體驗應像健康媒體，不像商城、診所、產品頁或政府宣導頁。

避免以下偏離品牌方向的設計與表現：

- 商品銷售感
- 醫療恐懼感
- 過度可愛插圖
- 誇張宣傳語
- AI 模板語氣
- 把保健食品變成全站主軸

## 2. 色彩使用原則

本階段僅根據既有 design tokens 記錄使用規則，不調整 token 值。

- `--color-paper`：主背景，大面積使用。
- `--color-ink`：主要文字。
- `--color-teal`：品牌識別、重點線條、active state。
- `--color-navy`：Footer、深色區塊。
- `--color-coral`：CTA 或少量提醒色，不可大面積使用。
- 分類色：只用於分類識別、badge、卡片頂線，不做大面積背景。
- verdict 色：只用於闢謠判讀，不挪作其他裝飾用途。

建議使用比例：

- Paper / white / neutral：70%
- Ink / body text：15%
- Teal / navy：10%
- Coral / alert / CTA：5% 以下

禁止事項：

- 任意新增 pastel hex
- 任意硬寫品牌外顏色
- 用過度鮮豔顏色製造保健品廣告感

## 3. 字體使用原則

> 本階段只記錄目前字體使用規則，不調整任何現有字體設定。

- `--font-serif`：品牌字、Hero 主標、部分媒體感標題。
- `--font-sans`：正文、文章標題、內容卡片標題。
- `--font-ui`：導覽列、badge、metadata、按鈕、介面文字。
- `--font-display`：特殊標題或視覺展示，應少量使用。

禁止事項：

- 未經討論更換全站字體
- 任意把所有 UI 改成 serif 或所有標題改成 sans
- 用外部 CDN 載入字體
- 上傳或分享字體檔

## 4. 卡片與元件語言

全站卡片應維持一致且可延伸的媒體風格：

- 圓角使用既有 radius token。
- 邊框使用 fog / token。
- 陰影使用既有 shadow。
- 不同內容類型用分類色做識別。
- 卡片不要有過多裝飾。
- 不要把每一頁做成完全不同的卡片語言。

適用元件包含：

- ArticleCard
- MythCard
- IngredientCard
- PodcastCard
- VideoCard
- NewsCard

卡片定位應像媒體內容入口，不像商品卡、課程銷售卡或電商卡。

## 5. 圖像語言

本日有據圖像風格應偏向以下方向：

- editorial
- abstract
- evidence-based
- clean
- calm
- health literacy

避免以下圖像方向：

- 商品攝影感
- 膠囊瓶罐銷售感
- 醫療恐嚇圖
- 過度寫實病灶圖
- 過度卡通
- 過度政府宣導感
- 十字架元素，除非使用者明確要求；一般召會、基督徒或健康內容設計也不要自動加入十字架

## 6. Logo 使用規範

- TopNav 使用 logo icon + 中文「本日有據」。
- Footer 可使用「本日有據 Evidence Today」。
- 一般頁面不要反覆展示大 Logo。
- Logo 不可拉伸變形。
- Logo 不可任意換色。
- Logo 不可加厚重陰影。
- Logo 周圍需保留安全距離。
- 橫式 Logo、圖標、中文名稱的使用場景需保持一致。

## 7. OG 圖規範

OG 圖是連結分享到 LINE、Facebook、Threads、X、Notion 等平台時顯示的預覽圖。

OG 圖應包含：

- 分類 badge
- 內容標題
- 本日有據 Evidence Today
- 發布日或更新日
- 適度使用分類色

OG 圖避免：

- 字太小
- 標題超出邊界
- 商品銷售視覺
- 過度醫療恐懼圖
- 與頁面分類色不一致
- 沒有品牌識別

## 8. 文案語氣

本日有據語氣應該是：

- 清楚
- 有根據
- 白話
- 保留不確定性
- 分清研究證據與主編判讀
- 先求準確，再求好懂

避免：

- AI 感套句
- 過度聳動
- 產品推銷語
- 只有口號，沒有資訊
- 把單一研究講成定論
- 把保健食品或任何單一主題變成全站中心
